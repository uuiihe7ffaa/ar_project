# Aspect Review Analyzer

Веб-приложение и API для определения **основного аспекта пользовательского отзыва** с использованием методов машинного обучения.

Проект позволяет автоматически анализировать текст отзыва и определять, о чём он в первую очередь:

* качество
* сервис
* цена
* атмосфера
* доставка

Система ориентирована на короткие пользовательские отзывы о заведениях (кафе, рестораны, доставка).

---

## Основные возможности

* Классификация отзыва по одному основному аспекту:

  * `качество`
  * `сервис`
  * `цена`
  * `атмосфера`
  * `доставка`

* Вычисление вероятностей по каждому аспекту

* Поддержка разных типов заведений:

  * кафе
  * ресторан
  * доставка

* API для интеграции

* Веб-интерфейс для быстрого тестирования

---

## Используемые технологии

* Python
* FastAPI
* scikit-learn
* LogisticRegression
* TF-IDF
* Streamlit

---

## Архитектура проекта

Проект разбит на несколько модулей:

* `training/train_model.py` — обучение модели
* `app/predictor.py` — загрузка модели и предсказание
* `app/main.py` — API (FastAPI)
* `app/schemas.py` — схемы запросов и ответов
* `ui/streamlit_app.py` — веб-интерфейс

Дополнительно:

* `training/dataset_single_aspect_150.csv` — датасет
* `model.pkl` — обученная модель
* `mlb.pkl` — encoder классов

## Принцип работы

1. Текст преобразуется в TF-IDF представление
2. Модель LogisticRegression:

   * анализирует текст
   * выбирает наиболее вероятный аспект
3. Возвращаются:

   * основной аспект
   * вероятности по всем классам

* Используется single-label классификация

## Реализация проекта в соответствии с методологией CRISP-DM

### 1. Постановка задачи (Business Understanding)

Целью данного проекта является разработка программного решения, способного автоматически определять основной аспект пользовательского отзыва. Под аспектом понимается ключевая характеристика, на которую обращает внимание пользователь: качество, сервис, цена, атмосфера или доставка.

Задача формализуется как задача классификации текста, где каждому отзыву необходимо сопоставить один наиболее релевантный класс. Практическая значимость проекта заключается в возможности применения системы для анализа обратной связи пользователей, мониторинга качества услуг и выявления проблемных зон в работе заведений.

---

### 2. Анализ данных (Data Understanding)

В качестве исходных данных используются короткие пользовательские отзывы о различных типах заведений, включая кафе, рестораны и сервисы доставки. Тексты характеризуются неформальным стилем, наличием разговорных выражений, сокращений и иногда грамматических ошибок.

Анализ показал, что многие отзывы содержат сразу несколько аспектов, однако в рамках данной реализации была выбрана стратегия выделения одного основного аспекта. Это упрощает задачу обучения модели и делает поведение системы более предсказуемым.

Также было замечено, что модель чувствительна к формулировкам, поэтому качество и разнообразие датасета напрямую влияет на итоговую точность классификации.

---

### 3. Подготовка данных (Data Preparation)

На этапе подготовки данных текстовые отзывы были приведены к единому виду: выполнена нормализация регистра, удалены лишние символы и приведена структура данных к табличному формату.

Каждому отзыву был присвоен один основной аспект, что позволило перейти от multi-label постановки задачи к single-label. Это решение позволило избежать "размытия" предсказаний и улучшить интерпретируемость результатов.

Для представления текста использовалась модель TF-IDF, позволяющая учитывать важность слов в контексте всей выборки. В результате каждый отзыв преобразуется в числовой вектор, пригодный для подачи в модель машинного обучения.

---

### 4. Моделирование (Modeling)

В качестве модели была выбрана Logistic Regression, которая хорошо подходит для задач классификации текстов и показывает стабильные результаты на небольших датасетах.

Модель обучается на TF-IDF признаках и предсказывает один из заранее заданных классов. Дополнительно вычисляются вероятности для каждого аспекта, что позволяет оценить степень уверенности модели.

Выбор простой модели обусловлен необходимостью получить быстрый, интерпретируемый и легко масштабируемый результат без усложнения архитектуры.

---

### 5. Оценка результатов (Evaluation)

Оценка качества модели проводилась с использованием метрики accuracy, а также путем анализа реальных предсказаний. Особое внимание уделялось случаям, когда модель ошибается или выбирает нерелевантный аспект.

В ходе тестирования было выявлено, что основная проблема заключается не в самой модели, а в качестве и структуре датасета. Слишком общие или неоднозначные отзывы приводят к снижению точности.

В результате было принято решение упростить формулировки в датасете и сделать их более однозначными, что позволило улучшить стабильность предсказаний.

---

### 6. Внедрение (Deployment)

На этапе внедрения была реализована полноценная система, включающая API и веб-интерфейс. API построено с использованием FastAPI и предоставляет endpoint для анализа текста, возвращая основной аспект и вероятности по всем классам.

Дополнительно реализован веб-интерфейс на Streamlit, позволяющий пользователю вводить текст отзыва и получать результат в удобной форме.

Таким образом, система может использоваться как самостоятельный инструмент анализа или как часть более сложных решений, связанных с обработкой пользовательской обратной связи.
